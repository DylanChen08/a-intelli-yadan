## 修复周对比表数据缺失 Bug（两次修复）

**问题现象**：近一周上下班对比表中，最近几天（05-31、06-01、06-02）显示"缺勤"，数据异常。

### 第一轮修复（14:51）- 误判
以为是 `pageSize: 100` 导致分页截断，将周查询改为 `pageSize: 500`。
- `src/record.js` - `queryRecords` 新增 `pageSize` 参数
- `src/weekly.js` - `getPersonWeekRecords` 传入 `pageSize: 500`

### 第二轮修复（15:04）- 找到真正根因
通过对比测试发现 **API 存在 bug**：
- 1天/2天查询 → 正常返回 06-01、06-02 数据
- 3天查询（含 05-31）→ **返回 0 条**
- 4~7天查询 → 只返回到 05-30，之后数据全丢

**根因**：MaxVision API 的 `queryIdentifyRecord` 接口，当查询范围包含没有任何通行记录的日期（05-31 周日）时，会导致该日期之后的所有数据丢失！

**真正修复**：`src/weekly.js` - `getPersonWeekRecords` 改为**按天循环单独查询**，每天查 `00:00:00~23:59:59`，然后合并记录。彻底避开 API 跨天查询 bug。
